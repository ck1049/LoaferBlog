# Loafer Blog - 功能更新产品需求文档

## 概述
- **摘要**：本次更新为 Loafer Blog 应用添加多个新功能和修复，包括数据库 schema 更新、全局配置管理、文件大小限制配置、RSA 密钥对管理、邮箱加密、默认头像生成和消息功能改进。
- **目的**：增强 Loafer Blog 应用的安全性、功能性和用户体验。
- **目标用户**：博客管理员和用户。

## 目标
- 更新所有表的 DDL，将 delete_time 默认值设置为 '00001-01-01 00:00:00'
- 添加全局配置表，至少包含以下字段：主键、模块、key、value、描述
- 实现文件大小限制配置，存储在配置表中，并根据该值动态修改 web 应用的上传文件大小限制
- 添加可持久化的业务 RSA 密钥对，用于后端小字段敏感数据的持久化存储
- 更新用户注册功能，将邮箱改为非必填项，并使用业务 RSA 公钥加密保存到数据库
- 在个人资料页面增加邮箱地址的回显与修改功能
- 修复发送消息功能的 sender_id 为空错误
- 为没有头像的用户生成默认头像
- 在消息页增加添加朋友功能，支持通过用户名搜索用户并分页查询

## 非目标（范围外）
- 更改现有的认证机制
- 修改未在需求中提及的其他现有功能
- 添加超出指定范围的新功能

## 背景与上下文
- 应用目前使用 MyBatis Plus 作为 ORM，PostgreSQL 作为数据库
- RSA 加密已用于前端-后端通信
- 应用具有文件上传功能，有大小限制
- 用户注册目前需要邮箱
- 用户可以相互发送消息
- 用户可以有头像

## 功能需求
- **FR-1**：更新所有表的 DDL，将 delete_time 默认值设置为 '00001-01-01 00:00:00'
- **FR-2**：添加全局配置表，包含字段：id、module、key、value、description
- **FR-3**：使用全局配置表实现文件大小限制配置
- **FR-4**：添加业务 RSA 密钥对管理，用于敏感数据加密
- **FR-5**：更新用户注册，将邮箱改为非必填项，并使用业务 RSA 公钥加密
- **FR-6**：在个人资料页面增加邮箱地址的回显与修改功能
- **FR-7**：修复发送消息功能的 sender_id 为空错误
- **FR-8**：为没有头像的用户生成默认头像
- **FR-9**：在消息页增加添加朋友功能，支持通过用户名搜索用户并分页查询

## 非功能需求
- **NFR-1**：安全性 - 所有敏感数据必须使用 RSA 加密
- **NFR-2**：性能 - 用户搜索和分页应该高效
- **NFR-3**：可靠性 - 应用应该优雅处理缺失的 RSA 密钥文件
- **NFR-4**：可维护性 - 代码应该遵循现有的模式和约定

## 约束
- **技术**：PostgreSQL 数据库，MyBatis Plus ORM，Spring Boot 框架
- **业务**：不应添加额外的依赖
- **依赖**：现有的 RSAUtils 工具用于生成 RSA 密钥

## 假设
- RSAUtils 工具已经存在，可以生成 RSA 密钥对
- 应用有用于头像的文件上传目录
- 应用有全局缓存机制

## 验收标准

### AC-1：删除时间默认值
- **给定**：所有表都有 delete_time 字段
- **当**：插入新记录时
- **则**：delete_time 字段自动设置为 '00001-01-01 00:00:00'
- **验证**：`programmatic`

### AC-2：全局配置表
- **给定**：应用正在运行
- **当**：查询配置表时
- **则**：表存在且包含所需字段
- **验证**：`programmatic`

### AC-3：文件大小限制配置
- **给定**：配置表中的文件大小限制已更新
- **当**：上传文件时
- **则**：上传根据配置的限制进行限制
- **验证**：`programmatic`

### AC-4：业务 RSA 密钥对管理
- **给定**：应用启动
- **当**：业务 RSA 密钥文件缺失时
- **则**：生成新的密钥文件并存储
- **验证**：`programmatic`

### AC-5：用户注册（邮箱可选）
- **给定**：用户注册时不提供邮箱
- **当**：提交注册
- **则**：用户创建成功
- **验证**：`programmatic`

### AC-6：邮箱加密
- **给定**：用户注册时提供邮箱
- **当**：邮箱存储到数据库时
- **则**：邮箱使用业务 RSA 公钥加密
- **验证**：`programmatic`

### AC-7：邮箱显示和修改
- **给定**：用户查看个人资料
- **当**：从数据库检索邮箱时
- **则**：邮箱被解密并脱敏后显示
- **验证**：`programmatic`

### AC-8：消息发送修复
- **给定**：用户发送消息
- **当**：提交消息
- **则**：消息保存时没有 sender_id 为空错误
- **验证**：`programmatic`

### AC-9：默认头像生成
- **给定**：用户没有头像
- **当**：应用启动时
- **则**：为用户生成默认头像并分配
- **验证**：`programmatic`

### AC-10：添加朋友功能
- **给定**：用户搜索其他用户
- **当**：提交搜索
- **则**：返回分页的用户列表
- **验证**：`programmatic`
