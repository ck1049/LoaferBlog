# 评论功能增强 - 产品需求文档

## Overview
- **Summary**: 增强帖子详情页的评论功能，包括显示评论用户的头像和用户名、优化评论时间显示、实现回复数展示和展开/折叠功能，并美化相关按钮样式。
- **Purpose**: 提升用户体验，使评论功能更加直观和美观。
- **Target Users**: 所有使用博客系统的用户。

## Goals
- 显示评论所属人的头像和用户名
- 优化评论时间的显示格式
- 默认只展示顶级评论
- 当评论有回复时，显示回复数和展开按钮
- 点击展开按钮查询并嵌套渲染回复
- 美化展开/折叠按钮的样式

## Non-Goals (Out of Scope)
- 不修改评论的发布和删除功能
- 不修改评论的分页加载逻辑
- 不修改其他页面的评论展示

## Background & Context
- 现有评论功能已经实现了基本的评论和回复功能
- 后端已经实现了评论的分页查询和回复查询接口
- 前端已经有基本的评论展示界面，但缺少用户头像、回复数显示和展开/折叠功能

## Functional Requirements
- **FR-1**: 评论应显示用户头像、用户名和格式化的评论时间
- **FR-2**: 默认只展示顶级评论，不自动加载回复
- **FR-3**: 当评论有回复时，显示回复数和展开按钮
- **FR-4**: 点击展开按钮应加载并显示该评论的回复
- **FR-5**: 点击折叠按钮应隐藏已加载的回复

## Non-Functional Requirements
- **NFR-1**: 回复的加载应平滑，避免页面卡顿
- **NFR-2**: 按钮样式应美观，与整体UI风格一致
- **NFR-3**: 响应式设计，在不同设备上都能正常显示

## Constraints
- **Technical**: 前端使用Vue 3 + Pinia，后端使用Spring Boot + MyBatis Plus
- **Dependencies**: 依赖后端的评论查询接口和用户信息

## Assumptions
- 后端API能够返回评论相关的用户信息（包括头像）
- 前端能够正确处理回复的加载和渲染

## Acceptance Criteria

### AC-1: 评论显示用户信息
- **Given**: 用户访问帖子详情页
- **When**: 页面加载完成
- **Then**: 每条评论应显示用户头像、用户名和格式化的评论时间
- **Verification**: `human-judgment`

### AC-2: 回复数和展开按钮
- **Given**: 评论有回复
- **When**: 评论加载完成
- **Then**: 评论下方应显示回复数和展开按钮
- **Verification**: `human-judgment`

### AC-3: 展开/折叠功能
- **Given**: 评论有回复
- **When**: 用户点击展开按钮
- **Then**: 应加载并显示该评论的回复
- **When**: 用户点击折叠按钮
- **Then**: 应隐藏已加载的回复
- **Verification**: `human-judgment`

### AC-4: 按钮样式美化
- **Given**: 用户访问帖子详情页
- **When**: 查看评论区域
- **Then**: 展开/折叠按钮应美观，与整体UI风格一致
- **Verification**: `human-judgment`
