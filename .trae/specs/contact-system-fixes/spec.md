# 联系人系统问题修复 - 产品需求文档

## Overview
- **Summary**: 修复联系人系统存在的多个问题，包括红点标记不显示、普通用户无法发送文件、通讯录页面报错、图标区分问题和移动端布局问题。
- **Purpose**: 解决用户反馈的功能缺陷，提升联系人系统的稳定性和用户体验。
- **Target Users**: 所有注册用户，包括普通用户和管理员。

## Goals
- 修复联系人列表的红点标记功能，确保未读消息能正确显示
- 解决普通用户无法发送文件的问题
- 修复通讯录页面的报错问题，确保页面正常加载
- 为通讯录按钮添加与消息按钮不同的图标
- 优化移动端布局，解决右侧留白过大的问题

## Non-Goals (Out of Scope)
- 不修改现有的消息系统核心功能
- 不添加新的功能特性
- 不修改后端的API接口设计

## Background & Context
- 联系人系统已经实现了基本功能，但存在多个用户反馈的问题
- 这些问题影响了用户体验，需要及时修复
- 修复应该保持与现有系统的兼容性

## Functional Requirements
- **FR-1**: 联系人列表显示未读消息红点标记
- **FR-2**: 普通用户可以发送文件消息
- **FR-3**: 通讯录页面正常加载，无报错
- **FR-4**: 通讯录按钮使用与消息按钮不同的图标
- **FR-5**: 移动端布局优化，左右留白平衡

## Non-Functional Requirements
- **NFR-1**: 页面响应速度快，加载时间不超过1秒
- **NFR-2**: 界面设计保持一致性，符合现有系统风格
- **NFR-3**: 功能操作流程简单直观，易于用户理解

## Constraints
- **Technical**: 基于现有Vue 3 + TypeScript + Spring Boot技术栈
- **Business**: 保持现有功能的兼容性，不破坏已有功能
- **Dependencies**: 依赖现有的用户认证和消息系统

## Assumptions
- 现有数据库结构和API接口保持不变
- 用户已登录并具有有效的认证状态
- 消息系统的基本功能正常运行

## Acceptance Criteria

### AC-1: 联系人列表红点标记
- **Given**: 用户收到新消息
- **When**: 用户查看联系人列表
- **Then**: 对应联系人卡片显示红点标记，标记数量与未读消息数一致
- **Verification**: `programmatic`
- **Notes**: 红点标记应在用户查看消息后消失

### AC-2: 普通用户发送文件
- **Given**: 普通用户登录系统
- **When**: 用户在聊天窗口上传并发送文件
- **Then**: 文件成功上传并发送，接收方可以查看和下载
- **Verification**: `programmatic`
- **Notes**: 确保文件上传权限正确配置

### AC-3: 通讯录页面加载
- **Given**: 用户点击通讯录按钮
- **When**: 通讯录页面加载
- **Then**: 页面正常显示好友列表，无报错
- **Verification**: `programmatic`
- **Notes**: 修复AddFriendView.vue中的trim()方法调用错误

### AC-4: 通讯录图标区分
- **Given**: 用户查看顶部导航栏
- **When**: 用户比较消息按钮和通讯录按钮
- **Then**: 两个按钮使用不同的图标，易于区分
- **Verification**: `human-judgment`
- **Notes**: 可以使用现有的其他图标或添加新图标

### AC-5: 移动端布局优化
- **Given**: 用户在移动端设备上访问系统
- **When**: 用户查看页面布局
- **Then**: 页面左右留白平衡，布局合理
- **Verification**: `human-judgment`
- **Notes**: 确保在不同屏幕尺寸上都有良好的显示效果
