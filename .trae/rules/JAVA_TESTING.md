# Java 测试规范

## 1. 单元测试
- **覆盖率**：核心方法必须测试，覆盖率≥70%，重点测试边界情况和异常场景
- **命名**：测试方法名格式为`test+业务方法名+场景`，如`testGetUserById_UserExist`
- **数据**：禁止操作生产数据，使用测试容器或内存数据库，测试数据应独立

## 2. 集成测试
- **范围**：测试模块间交互、与外部系统集成、数据库操作和事务处理
- **环境**：使用独立的测试环境配置，与生产环境保持相似

## 3. 最佳实践
- **原则**：可重复性、独立性、明确的断言
- **工具**：JUnit 5、Mockito、TestContainers、AssertJ、Spring Test
- **结构**：测试代码与生产代码结构一致，测试类命名为`被测试类名+Test`

## 4. 持续集成
- **CI/CD**：集成到流水线，每次提交自动运行测试，失败时阻止合并
- **报告**：生成覆盖率报告，分析结果，持续改进测试质量
