# Java 测试规范

## 1. 单元测试

### 1.1 测试覆盖率
- 核心业务方法必须编写单元测试（使用JUnit 5）
- 测试覆盖率不低于70%
- 重点测试边界情况和异常场景

### 1.2 测试命名
- 测试方法名格式为`test+业务方法名+场景`
- 示例：`testGetUserById_UserExist`

### 1.3 测试数据
- 禁止操作生产环境数据
- 使用测试容器或内存数据库
- 测试数据应独立，避免测试间相互影响

## 2. 集成测试

### 2.1 测试范围
- 测试模块间的交互
- 测试与外部系统的集成
- 测试数据库操作和事务处理

### 2.2 测试环境
- 使用独立的测试环境配置
- 测试环境应与生产环境保持相似

## 3. 测试最佳实践

### 3.1 测试原则
- 测试应具有可重复性
- 测试应具有独立性
- 测试应具有明确的断言

### 3.2 测试工具
- JUnit 5：单元测试框架
- Mockito：模拟对象
- TestContainers：测试容器
- AssertJ：断言库
- Spring Test：Spring集成测试

### 3.3 测试代码结构
- 测试代码应与生产代码结构一致
- 测试类命名为`被测试类名+Test`

## 4. 持续集成

### 4.1 CI/CD集成
- 集成到CI/CD流水线中
- 每次提交自动运行测试
- 测试失败时阻止代码合并

### 4.2 测试报告
- 生成测试覆盖率报告
- 分析测试结果
- 持续改进测试质量
