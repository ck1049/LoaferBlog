# Java 代码质量规范

## 1. 数据类型与空值处理

### 1.1 基本类型
- 优先使用包装类（如`Integer`），避免NPE
- 集合/数组必须判空后使用

### 1.2 字符串处理
- 判空使用`StringUtils.isEmpty()`/`StringUtils.isBlank()`
- 避免使用`==`比较字符串
- 优先使用`StringBuilder`进行字符串拼接

## 2. 异常处理

### 2.1 异常捕获
- 禁止捕获`Exception`大而全的异常，需捕获具体异常
- 禁止在catch块中吞异常（至少打印日志）

### 2.2 自定义异常
- 自定义业务异常，统一返回异常码和信息
- 异常信息应清晰明确，便于问题定位

## 3. 集合与流操作

### 3.1 集合初始化
- 指定初始容量（如`new HashMap<>(16)`），避免扩容损耗
- 选择合适的集合类型

### 3.2 Stream流操作
- 避免嵌套流操作，复杂逻辑拆分
- 终止操作必须添加，禁止无终止的流

## 4. 版本与依赖规范

### 4.1 JDK版本
- 统一使用JDK 11/17

### 4.2 依赖管理
- 核心依赖在`pom.xml`中统一版本号
- 禁止引入未使用的依赖
- 使用依赖管理工具统一管理版本
